PROJECT OVERVIEW
────────────────
Kravon is a restaurant SaaS platform with:
  • kravon-backend: Fastify.js API server (Node.js)
  • kravon-frontend: Vanilla HTML/CSS/JS storefront (zero frameworks, zero build tools)
  • kravon-db: PostgreSQL schema + migrations
  • Database: Railway PostgreSQL (turntable.proxy.rlwy.net:17128)

✅ Backend: Running, connected to Railway DB, serving mock data
✅ Frontend: Implemented, schema-driven design (all values from API)
✅ Database: Connected and accessible
✅ Git: Initial commit done, ready for push

QUICK START
───────────

1) BACKEND (Terminal 1)
   cd kravon-backend
   npm install
   npm run dev
   
   Expected output:
     [DB] Connected to PostgreSQL — server time: ...
     Server listening at http://127.0.0.1:3002
     Health check → http://localhost:3002/health

2) FRONTEND (Terminal 2)
   cd kravon-frontend
   python -m http.server 8000
   
   Then open in browser:
     http://localhost:8000/presence/index.html

3) TEST ENDPOINTS
   Health:   curl http://localhost:3002/health
   Presence: curl http://localhost:3002/api/presence/spice-of-india

KNOWN ISSUES & NOTES
────────────────────

1) PORT 3002 "EADDRINUSE" on restart:
   Root cause: Old Node process still bound to port
   Fix (cmd.exe):
     netstat -ano | findstr :3002          (find PID)
     taskkill /PID <pid> /F                (kill process)
   Fix (PowerShell):
     Stop-Process -Id (Get-NetTCPConnection -LocalPort 3002).OwningProcess -Force
   Then: npm run dev

2) Frontend spinner stuck:
   Root cause: Backend not running or API request timed out
   Fix:
     - Check backend is running: curl http://localhost:3002/health
     - Open browser DevTools → Console and Network tabs for errors
     - Verify .env has correct DB credentials

3) Database connection timeout:
   Root cause: Railway proxy connection or network firewall
   Fix:
     - Test standalone: node kravon-backend/test-db.js
     - Check Railway dashboard for DB status
     - Verify IP whitelist on Railway if applicable
