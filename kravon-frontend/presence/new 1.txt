**Standard target structure** (adapt as needed based on your analysis):

```
[business-name]/
```

If the page has meaningful extra complexity — multi-step ordering, booking flows, tab navigation, modals, filters — add the JS and CSS files that complexity warrants. Name them for what they do. Don't shoehorn everything into the standard 5 JS files if it doesn't fit.

---

**Non-negotiable rules regardless of complexity:**

**Config:** Everything restaurant-specific lives in `config/config.js` only. No hardcoded names, numbers, copy, prices, or phone numbers anywhere else. Structure CONFIG with logical sections matching the page's actual content. MENU items always have string IDs.

**HTML:** Zero hardcoded content. Use:
- `data-bind="config.path"` for innerHTML (rich text, headlines)
- `data-bind="config.path" data-bind-text` for textContent (plain strings)
- `data-wa-link` on all WhatsApp anchors
- Named IDs on every container JS fills from an array
- SVG WhatsApp icon as a single `<symbol>` sprite, referenced with `<use href="#icon-wa">`
- Scripts at the bottom in dependency order (config first, main last)

**JS:** No inline `onclick`. All interactions via `data-action` / `data-id` attributes handled by a single delegated click listener in `main.js`. Each module is an IIFE. No globals. `cart.js` is pure logic with no DOM access. Item IDs always treated as strings. Null-guard any `MENU.find()` call. `window.open()` always uses `'noopener,noreferrer'`.

**CSS:** No inline styles except `display:none` on elements toggled by JS. All colours as CSS custom properties in `:root`. No hardcoded hex values outside of `base.css`. Responsive rules consolidated in one place.

**Security:** All `target="_blank"` links have `rel="noopener noreferrer"`. No `innerHTML` injection from any data that could come from user input or external sources.

4. Check every `data-action` value in rendered HTML has a matching `case` in the click handler.
5. Check script load order — every module must be defined before any module that depends on it.
6. Check CSS brace balance in every file.

Fix anything the audit finds before delivering. Report what the audit found and what was fixed.
